Claude Code Modes + Best Pairings
Plan Mode
→ pair with /plan
Thinks before acting. /plan adds the planner framework on top.
Edit Mode
→ pair with /tdd · /build-fix · /refactor-clean
Writes code. These commands control how it writes.
Ask Mode
→ pair with /docs · /code-review · /aside
Answers questions. These give it specialized knowledge.
Everyday Commands — Use on Every Project
/plan
Plan Before Coding
Structured phases, dependencies, risk analysis, testing strategy, success criteria. Waits for approval before any code is written.
/plan Add a REST API for user management with auth
/tdd
Test-Driven Development
Writes failing tests first (RED), implements minimal code (GREEN), then refactors. Enforces 80%+ coverage. JS/TS default.
/tdd Write a utility that parses CSV files with error handling
/code-review
Security & Quality Review
Scans staged/unstaged changes. Severity: CRITICAL HIGH MED LOW
/code-review
/build-fix
Fix Build Errors
Auto-detects build system (npm, cargo, go, gradle, maven…). Incrementally fixes errors with minimal changes.
/build-fix
/e2e
End-to-End Testing
Generates Playwright tests, runs across browsers, captures screenshots / videos / traces.
/e2e Test the complete signup-to-dashboard flow
/verify
Pre-Merge Verification
Comprehensive checks: build · type check · lint · test validation. Run before every merge.
/verify
